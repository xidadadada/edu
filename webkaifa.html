前端主要基于Vue、React、Angular和Jquery这四大框架衍生。


jquery

项目功能比较简单。并不需要做成一个单页面应用。


React

Facebook开发的不是一个MVC框架，构建易于可重复调用的web组件，侧重于UI, 也就是view层。是单向的从数据到视图的渲染，非双向数据绑定。不直接操作DOM对象，而是通过虚拟DOM通过diff算法以最小的步骤作用到真实的DOM上。


angular

google开发的基于数据驱动, 适合做数据操作比较繁琐的项目,angular适用于单页面开发,是一个比较完善的mvvm框架, 包含模板和双向数据绑定, 路由, 模块化, 服务, 过滤器, 依赖注入等所有功能,但是angular验证功能比较薄弱。


vue

比angular更轻量, 性能上更高效, 比angular更容易上手。


Backbone

一个轻量级的前端MVC框架，用于结构化管理页面中的大量JS，建立与服务器、视图间的无缝连接。


Flux

Facebook开发的前端架构， 利用一个单向的数据流补充了React的组合视图组件。


Ember

用于创建 web 应用的 JavaScript MVC 框架，采用基于字符串的Handlebars模板，支持双向绑定、观察者模式、计算属性（依赖其他属性动态变化）、自动更新模板、路由控制、状态机等。 


Bootstrap 

Twitter 开发的CSS/HTML框架。全局的 CSS设置、定义基本的 HTML 元素样式、可扩展的 class，以及一个先进的网格系统。包含了十几个可重用的组件，用于创建图像、下拉菜单、导航、警告框、弹出框等等。包含了十几个自定义的jQuery 插件。


DOM


可以把 DOM 看成节点，可以使用 Javascript 语言来操作 DOM 以改变网页。

    Git工作流：

    • 集中式工作流：跟svn类似，它只有一个master分支，比较适合小团队

    • 功能开发工作流：从master拉取feature分支进行功能开发，功能开发完成后，会向master分支发起Pull Request，只有审核通过的代码才允许合入。

    • Gitflow工作流：master和develop分支是一直存在的，master分支可以被视为稳定的分支，而develop分支是相对稳定的分支，特性开发会在feature分支上进行，发布会在release分支上进行，而bug修复则会在hotfix分支上进行。

    • Forking工作流：有一个公开的中央仓库，其他贡献者可以Fork（克隆）这个仓库作为你自己的私有仓库，开源项目维护者可以直接往中央仓库push代码，而代码贡献者只能将代码push到自己的私有仓库。


    Webpack：把你的项目当做一个整体，通过一个给定的主文件（如：index.js），Webpack将从这个文件开始找到你的项目的所有依赖文件，使用loaders处理它们，最后打包为一个（或多个）浏览器可识别的JavaScript文件。

    Grunt和Gulp：在一个配置文件中，指明对某些文件进行类似编译，组合，压缩等任务的具体步骤，工具之后可以自动替你完成这些任务。

    后端

    node.js

    通过JavaScript语言开发web服务端，有非阻塞，事件驱动I/O等特性，从而让高并发（high concurrency）在的轮询（Polling）和comet构建的应用中成为可能。v8引擎是由Google推出的开源JavaScript引擎。得益于JIT，编译模式的改变与编译阶段的优化，JavaScript的性能得到了一个飞跃。其源代码是用c++写的。


    长轮询

     long polling 是一种利用HTTP模拟持续连接的技巧。只要页面载入了，不管需不需要服务器给你响应信息，你都会给服务器发一个Ajax请求。这个请求服务器不会直接给你返回信息，而是等服务器觉得该给你发信息了，它才会响应。虽然这些进程闲着，但是照样占用内存。


    非阻塞

    服务器一个loop循环，一个新请求来了，这个loop就把这个请求传给其他的进程（比如传给一个搞数据库查询的进程），然后响应一个回调（callback）。这loop就接着接其他的请求。


    事件驱动

    结果返回来了，loop就把结果传回用户的浏览器，接着继续跑。服务器只在用户那边有事件发生的时候才响应。
